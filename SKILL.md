# 日本声优年收入估算调研 Skill

## 适用场景

当用户要求估算日本声优、动画歌手、2.5D 声优艺人、企划型乐队成员、广播节目主持人的年收入时，使用本 Skill。

典型问题：

- “估算某某声优 2025 年年收入”
- “从动画、live、事务所抽成、IP 分成角度分析收入”
- “某声优靠某个 IP 一年大概赚多少钱”
- “把声优收入拆成长期饭票和短期业务”
- “做一个可复核的收入模型，而不是八卦传闻”

目标不是给出绝对真实收入，而是基于公开资料构建一个可复核、可替换参数、可解释不确定性的估算模型。

---

## 输出口径

默认输出口径为：

> 事务所抽成后、个人所得税和社保前的个人税前收入，单位为日元。

必须明确区分：

1. **模型总签约额 / gross contract value**  
   指项目方、制作方、节目方、品牌方等支付给事务所或 talent 侧的估算总额。

2. **事务所抽成后个人税前收入 / pre-tax personal income after agency cut**  
   指扣除事务所、经纪管理、制作代理等抽成后的个人税前收入。

3. **税后到手**  
   除非用户明确要求，否则不要估算。日本个人所得税、住民税、社保、个体经营扣除、法人化安排高度不透明。

---

## 核心原则

### 1. 公开可验证与模型假设必须分开

所有结论必须拆成两层：

- **公开事实**：官网 profile、作品 cast、live 日期、票价、会场容量、月费价格、商品定价、节目页、官方 X、ORICON、Animate Times、事务所页面等。
- **模型假设**：单话报酬、二次使用系数、live 出演费、平台分成、事务所抽成、付费转化率、IP bonus 等。

不要把假设写成事实。

### 2. 优先使用官方来源

证据优先级：

1. 事务所官网：所属、代表作、profile。
2. 作品 / IP 官网：cast、角色名、播出年份、live、公演、商品。
3. 官方票务页 / 会场页：票价、容量、日期。
4. 官方广播 / 节目 / 会员页：月费、放送频率、付费内容。
5. 可靠媒体：ORICON、Animate Times、Natalie、PR Times、Seiyu Awards 等。
6. 行业价格参考：日俳连、声优学校、casting 公司、广告代理 / 制作公司公开报价、CrowdWorks 等。
7. Wiki、萌娘百科、粉丝整理站只能用于补充线索，不作为核心收入证据。

### 3. 必须输出保守 / 基准 / 乐观三情景

不要只给一个单点数字。默认输出：

| 情景 | 模型总签约额 | 事务所抽成假设 | 个人税前收入 |
|---|---:|---:|---:|
| 保守 | ¥X | 30% | ¥X |
| 基准 | ¥X | 25% | ¥X |
| 乐观 | ¥X | 20% | ¥X |

如果事务所抽成有更强证据，可替换默认值。

### 4. IP 分成默认设为 0，除非有公开证据

声优通常是 cast / performer / singer / program personality，而不是制作委员会出资方。

因此：

- 保守：IP 权益分成为 0。
- 基准：IP 权益分成为 0。
- 乐观：可给小额 bonus，但必须说明这是尾部可能性，不是默认权益。

不要把角色周边销售额、CD 销售额、票房总流水直接当成声优个人收入。

## 调研流程

### Step 1：锁定人物身份

收集：日文姓名、罗马字 / 读法、事务所、出身地、生日等 profile、代表角色、奖项、是否有个人音乐活动、是否属于 project-live / real band / 企划乐队。

优先来源：事务所官网、Seiyu Awards 官网、官方作品页。

### Step 2：整理目标年份活动清单

按年份梳理目标年内可验证活动：TV 动画、剧场版、OVA、游戏角色、外画 / 吹替、广播 / 网络节目、live、concert、festival、talk event、fan meeting、stage greeting、角色歌、CD、OST、streaming、联名商品、voice 商品、会员订阅、CM、旁白、广告、品牌合作。

建议表：

| 年份 | 项目 | 类型 | 角色 / 身份 | 公开证据 | 可量化变量 |
|---|---|---|---|---|---|
| 2025 | 动画 A | TV 动画 | 主役 / 配角 | 官网 cast 页 | 话数 |
| 2025 | Live B | live | performer | 官网 event 页 | 日期、票价、会场 |
| 2025 | 节目 C | 广播 | 主持 | 官方节目页 | 放送频率、月费 |

### Step 3：区分长期饭票与短期业务

长期饭票指持续带来曝光、活动、音乐、商品化、会员、复演机会的收入源。常见类型包括长寿 IP 角色、BanG Dream! / LoveLive! / Idolmaster 类 project-live、常设广播节目、角色歌 catalog、会员订阅节目、持续商品化角色、高复用游戏角色。

短期业务指一次性或低复用收入。常见类型包括单话 guest、动画客串、stage greeting、节目 guest、宣传直播、单次旁白、单次广告 voice、一次性联名录音、中小型 talk event。

## 收入流建模

### 1. 动画 / 外画配音

```text
动画 gross = 话数等价单元 × 每单元首轮价 × 二次使用系数
```

| 情景 | 话数等价单元 | 首轮单价 | 二次使用系数 |
|---|---:|---:|---:|
| 保守 | 实际确认话数 × 0.8 | ¥50,000–¥60,000 | 1.5–1.6 |
| 基准 | 实际确认话数 | ¥70,000–¥85,000 | 1.8–2.0 |
| 乐观 | 实际确认话数 + 宣传 / 剧场 / 客串补正 | ¥90,000–¥120,000 | 2.0–2.3 |

注意：新人声优与资深声优价格差异很大；日俳连 rank 制与二次使用报酬是动画 / 外画估算核心；对上升期声优可高于纯新人，但不要直接套顶流 veteran 价格。

### 2. 游戏录音

```text
游戏 gross = 录音批次数 × 单批次估算价
```

| 情景 | 批次数 | 单批次价格 |
|---|---:|---:|
| 保守 | 1–2 | ¥100,000–¥200,000 |
| 基准 | 3–5 | ¥200,000–¥400,000 |
| 乐观 | 5–8+ | ¥400,000–¥800,000 |

手游角色如果长期更新语音，价值显著高于一次性游戏；如果只有角色名、没有可验证更新频率，使用 bundle 估算。

### 3. 广播 / 网络节目

```text
广播 gross = 放送回数 × 单回出演费
```

| 情景 | 单回出演费 |
|---|---:|
| 保守 | ¥5,000–¥15,000 |
| 基准 | ¥15,000–¥30,000 |
| 乐观 | ¥30,000–¥50,000 |

节目类型修正：单人冠名节目 > 多人节目；视频付き节目 > 纯音频节目；有赞助节目 > 无赞助节目；有会员 archive / after talk > 普通免费节目。

### 4. 会员订阅

```text
会员净分配 = 粉丝规模 proxy × 付费转化率 × 月费 × 有效月数 × talent pool 比例
```

| 情景 | 付费转化率 | talent pool 比例 |
|---|---:|---:|
| 保守 | 1%–2% | 5%–10% |
| 基准 | 2%–4% | 10%–15% |
| 乐观 | 5%–8% | 15%–25% |

月费总流水不是声优收入。平台、节目制作方、版权方、事务所都可能先分账。

### 5. Live / concert / festival

优先使用出演费口径，不直接使用票房分成。

```text
live gross = 大型 live 场次 × 大型单日出演费
           + 中小型 event 场次 × 中小型出演费
           + 配信 / 物贩 / bonus
```

| 类型 | 保守 | 基准 | 乐观 |
|---|---:|---:|---:|
| 大型 franchise live | ¥200,000 / 日 | ¥400,000–¥600,000 / 日 | ¥700,000–¥1,000,000 / 日 |
| Zepp / 中型 tour | ¥150,000 / 日 | ¥250,000–¥400,000 / 日 | ¥500,000+ / 日 |
| Talk event / 小型节目活动 | ¥50,000 / 场 | ¥80,000–¥200,000 / 场 | ¥200,000–¥400,000 / 场 |
| Festival | ¥100,000 / 日 | ¥200,000–¥400,000 / 日 | ¥500,000+ / 日 |

声优 performer 通常不是主办方，不应默认吃票房大头；如果是 project band 成员，固定出演费 + bonus 比票房分成更合理。

### 6. 角色歌 / 音乐

```text
音乐 gross = 录音费 + catalog / streaming / CD / live bonus
```

| 情景 | 年度音乐收入 |
|---|---:|
| 保守 | ¥100,000–¥300,000 |
| 基准 | ¥500,000–¥1,000,000 |
| 乐观 | ¥1,000,000–¥3,000,000+ |

如果只是角色歌，可能是一次性 buyout；如果本人是 artist 合约，可能有更强 royalty；如果参与作词 / 作曲，可另计版权收入。

### 7. 联名商品 / 周边 / voice 商品

```text
联名 gross = SKU 数 × 商品规模修正 × talent voice / promotion fee
```

或：

```text
联名 gross = 商品售价 × 估计销量 × talent pool 比例
```

| 情景 | 单项联名 / voice 商品 |
|---|---:|
| 保守 | ¥50,000–¥200,000 |
| 基准 | ¥200,000–¥600,000 |
| 乐观 | ¥600,000–¥1,500,000 |

普通角色周边收益属于 IP 权利方，不默认归声优。声优本人录制语音、拍摄宣传、节目联动商品，才可归入个人收入模型。

### 8. CM / 旁白 / 广告

```text
广告 gross = 项目数 × 单项目价格
```

| 情景 | 单项目价格 |
|---|---:|
| 保守 | ¥50,000–¥150,000 |
| 基准 | ¥150,000–¥600,000 |
| 乐观 | ¥600,000–¥2,000,000 |

全国级 TV CM 与普通 web CM 差异极大。声优如果只是语音出演，不等于 celebrity ambassador。

### 9. IP 权益分成

默认：

```text
IP equity / production committee participation = 0
```

只有在公开披露本人或事务所参与制作委员会、本人是 artist project 的核心权利人、本人拥有词曲版权、本人开设个人品牌或自营 merch、明确存在 profit participation / royalty 时才建模。

乐观情景可设置：

```text
IP bonus = ¥300,000–¥2,000,000
```

但必须标记为低置信度。

---

## 事务所抽成

| 情景 | 事务所抽成 |
|---|---:|
| 保守 | 30% |
| 基准 | 25% |
| 乐观 | 20% |

```text
个人税前收入 = gross 合计 × (1 - 事务所抽成率)
```

不同事务所、业务线、资历、合约形态差异很大。青二、事务所、唱片公司、制作公司、平台方可能分别参与分配。若无公开证据，不要宣称具体合同分成。

## 建议输出结构

### 1. 执行摘要

包含最终区间、最可能区间、基准点估计、最大收入来源、最大不确定性。

```text
我的结论是：该声优在目标年份的个人税前收入大概率位于 ¥X–¥Y，基准估计约 ¥Z。若把极端保守和乐观情景纳入，包络区间为 ¥A–¥B。收入主引擎是动画 / project-live / 广播节目，会员和周边是补充项，不是决定项。
```

### 2. 人物与商业化画像

说明声优处于新人、上升期、成熟期还是顶流期；是否有长期 IP、project-live、个人节目、音乐 / 角色歌、品牌合作。

### 3. 公开活动清单

| 项目 | 类型 | 公开事实 | 收入意义 |
|---|---|---|---|
| 动画 A | TV 动画 | 2025 年播出，担任某角色 | 动画台本费 + 二次使用 |
| Live B | live | 某日期某会场 | 固定出演费 + bonus |
| 节目 C | 广播 | 每周放送，月费 ¥X | 节目出演费 + 会员分配 |

### 4. 收入流估算表

| 收入流 | 保守 | 基准 | 乐观 |
|---|---:|---:|---:|
| 动画 / 剧场配音 | ¥X | ¥X | ¥X |
| 游戏 / 外画 / 旁白 | ¥X | ¥X | ¥X |
| 广播 / 节目 | ¥X | ¥X | ¥X |
| 会员订阅 | ¥X | ¥X | ¥X |
| live 固定出演费 | ¥X | ¥X | ¥X |
| live 分成 / 配信 / 物贩 bonus | ¥X | ¥X | ¥X |
| 音乐 / 角色歌 | ¥X | ¥X | ¥X |
| 联名 / 周边 / 商品化 | ¥X | ¥X | ¥X |
| IP 权益分成 | ¥0 | ¥0 | ¥X |
| gross 合计 | ¥X | ¥X | ¥X |
| 事务所抽成后个人税前 | ¥X | ¥X | ¥X |

### 5. 敏感性分析

必须指出最影响结果的变量：动画话数等价单元、动画单话价格、二次使用系数、live 单日出演费、是否有大型游戏角色、是否有广告 / CM、事务所抽成率。

| 变量变化 | 对个人税前收入影响 |
|---|---:|
| 动画平均单价 ±¥10,000 | ±¥X |
| 新增 / 减少一个 12 话常规角色 | ±¥X |
| live 单日出演费 ±¥100,000 | ±¥X |
| 事务所抽成 20%–30% 摆动 | ±¥X |
| 会员转化率 ±2pct | ±¥X |

### 6. 置信度

默认给出：

```text
置信度：中低。
```

原因：公开活动量可验证；公开价格和票价可验证；私人合同、事务所抽成、二次使用分配、平台会员分账、IP 权益通常不可验证。

## 计算模板

### 动画

```text
anime_gross_low  = anime_units_low  × anime_fee_low  × residual_low
anime_gross_base = anime_units_base × anime_fee_base × residual_base
anime_gross_high = anime_units_high × anime_fee_high × residual_high
```

### Live

```text
live_gross =
  large_live_days × large_live_day_fee
+ mid_live_days × mid_live_day_fee
+ small_event_count × small_event_fee
+ streaming_merch_bonus
```

### 会员

```text
membership_gross =
  fan_proxy
× paid_conversion_rate
× monthly_fee
× active_months
× talent_pool_share
```

### 最终收入

```text
gross_total =
  anime_gross
+ game_dubbing_gross
+ radio_program_gross
+ membership_gross
+ live_gross
+ music_gross
+ merch_collab_gross
+ cm_narration_gross
+ ip_bonus

personal_pre_tax_income =
  gross_total × (1 - agency_cut_rate)
```

## 注意事项

1. 不要用“粉丝觉得她很红”直接推收入。
2. 不要把 IP 总流水、票房、CD 销售额直接当个人收入。
3. 不要假设声优拥有制作委员会权益。
4. 不要使用未经核实的论坛年收传闻作为核心依据。
5. 不要把税后、税前、事务所前、事务所后混在一起。
6. 不要过度精确到个位数或十万日元级别。建议四舍五入到 10 万或 100 万日元。
7. 对上升期声优，动画和 project-live 通常比会员订阅更关键。
8. 对顶流声优，CM、旁白、个人 artist、品牌合作可能显著抬高收入。
9. 对纯新人，广播 / 动画单价要明显下修。
10. 对 solo artist，音乐和 live 权重要上修。

## 核心一句话

这套 Skill 的本质是：

> 公开活动量 × 行业价基 × 分账假设 × 三情景敏感性分析。

它适合估算量级，不适合声称精确年收。
